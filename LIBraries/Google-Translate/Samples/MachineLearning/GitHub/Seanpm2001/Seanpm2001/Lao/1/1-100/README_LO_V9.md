## ການ ນຳ ໃຊ້ README

** ເອກະສານ README ນີ້ປະຈຸບັນມີຈຸດປະສົງເພື່ອເປັນປະຕູສູ່ທຸກສິ່ງທີ່ຂ້ອຍສະ ເໜີ ຜ່ານທາງອິນເຕີເນັດ. ມັນເປັນວຽກທີ່ ກຳ ລັງ ດຳ ເນີນຢູ່, ແລະຍັງບໍ່ທັນຄົບຖ້ວນເທື່ອ. ມັນຍັງເປັນບາງສິ່ງບາງຢ່າງທີ່ຈະຕ້ອງໄດ້ຮັບການປັບປຸງເທື່ອລະກ້າວໃນທຸກໆຄັ້ງໃນເວລາດຽວກັນ. **

! [ຮູບພາບຂອງ GitHub Profile ລົ້ມເຫລວໃນການໂຫລດ. ກົດບ່ອນນີ້ເພື່ອພະຍາຍາມເບິ່ງມັນ] (SeniorPhotoFullQuality.jpeg)

ຮູບປະຈຸບັນຮອດວັນທີ 4 ມີນາ 2021 [ກົດເຂົ້າໄປທີ່ນີ້ເພື່ອດາວໂຫລດດ້ວຍຄຸນນະພາບ (JPEG)] (SeniorPhotoFullQuality.jpeg)

ຖ້າທ່ານໄດ້ຖືກສົ່ງມາທີ່ນີ້ຜ່ານທາງລິງຜ່ານທາງ SMS (ການສົ່ງຂໍ້ຄວາມ) ຊົມເຊີຍ! ທ່ານເປັນເພື່ອນສະ ໜິດ ສະ ໜົມ, ແລະ ໜຶ່ງ ໃນ ຈຳ ນວນ 73 ຄົນ / ບອທ໌ທີ່ເຂົ້າເບິ່ງເບີໂທລະສັບຂອງຂ້ອຍແລະໄດ້ຮັບການອະນຸມັດຈາກຂ້ອຍ. ຖ້າທ່ານຕ້ອງການ, ທ່ານສາມາດເບິ່ງທີ່ [ຂໍ້ແນະ ນຳ ກ່ຽວກັບການສົ່ງຕໍ່ຂໍ້ມູນຂອງຂ້ອຍ] (https://github.com/seanpm2001/SMS-Messaging-with-Sean), ຫຼືເວົ້າງ່າຍໆວ່າ: ຢ່າຂີ້ເຫຍື້ອ, ຢ່າ ' ບໍ່ບອກເບີໂທລະສັບຂອງຂ້ອຍ, ແລະຮູ້ສຶກບໍ່ເສຍຄ່າທີ່ຈະສົ່ງຂໍ້ຄວາມຫາຂ້ອຍແລະສົ່ງຂ້ອຍທຸກເວລາ (ແມ້ແຕ່ເວລາທ່ຽງຄືນ, ຫລື 3: 22) ຂ້ອຍອາດຈະບໍ່ຕອບຮັບທັນທີ, ແຕ່ຂ້ອຍຈະໄປຫາເຈົ້າເມື່ອຂ້ອຍຕື່ນ.

ນີ້ແມ່ນບັນຊີລາຍຊື່ຂອງໂຄງການທີ່ ສຳ ຄັນແບ່ງຕາມປະເພດໂຄງການຂະ ໜາດ ໃຫຍ່.

### ໂຄງການຊີວິດຂອງທ່ານ

[Degoogle your life] (https://github.com/seanpm2001/Degoogle-your-life) - ຫໍໄຕຫຼັກໃນຊຸດໂຄງການໂຄສະນາຕໍ່ຕ້ານ Google ຂອງຂ້ອຍ. ໂຄງການນີ້ພົວພັນກັບພວກເຂົາທັງ ໝົດ, ແລະໃຫ້ຂໍ້ມູນທົ່ວໄປ.

[ເປັນຫຍັງທ່ານຄວນຢຸດໃຊ້ Google Chrome] (https://github.com/seanpm2001/Why-you-should-stop-using-Chrome) - ບົດຂຽນ ທຳ ອິດໃນບົດຂຽນ degoogle ຂອງຂ້າພະເຈົ້າ, ເພື່ອແນໃສ່ແກ້ໄຂບັນຫາກັບ Chrome ແລະໃຫ້ທາງເລືອກທີ່ເຄົາລົບຄວາມເປັນສ່ວນຕົວທີ່ດີ.

[ຢຸດໃຊ້ Chromebook] (https://github.com/seanpm2001/Stop-using-Chromebooks) - ບົດຂຽນທີສອງໃນຊຸດຂອງບົດຂຽນ degoogle ຂອງຂ້ອຍ, ເພື່ອແນໃສ່ຊີ້ໃຫ້ເຫັນບັນຫາເພີ່ມເຕີມກັບ Google Chrome, ເວລານີ້ກັບ ChromeOS ແລະເດັ່ນ. ຄວາມເປັນສ່ວນຕົວ - ອຸປະກອນຄອມພິວເຕີ້ແລະແບບຄອມພິວເຕີ້ທີ່ມີຈຸດປະສົງເພື່ອເກັບ ກຳ ຂໍ້ມູນຜູ້ໃຊ້ຫຼາຍຂື້ນແລະເພີ່ມຂື້ນ Google monopoly. ທາງເລືອກຕ່າງໆກໍ່ໄດ້ຮັບເຊັ່ນກັນ, ທຸກໆບົດຂຽນໃຫ້ທາງເລືອກ.

[ອີກທາງເລືອກ ໜຶ່ງ ຈາກ YouTube] (https://github.com/seanpm2001/Alternating-from-YouTube) - ອີກ ໜຶ່ງ ບົດຂຽນໃນຊຸດບົດຂຽນ degoogle ຂອງຂ້ອຍ, ເພື່ອແນໃສ່ສະແດງຄວາມ ໜ້າ ຊື່ໃຈຄົດແລະອຸດົມສົມບູນຂອງບັນຫາ (ແລະມີຜູ້ເສຍຊີວິດຫຼາຍຄົນ + ຍິງ 1 ຄົນ) ເກີດຈາກ YouTube ທີ່ເປັນເຈົ້າຂອງ Google. ເນື່ອງຈາກວ່າມັນຍາກທີ່ຈະ ໜີ ຈາກ YouTube ເນື່ອງຈາກບໍ່ມີທາງເລືອກທີ່ ເໝາະ ສົມ, ບົດຄວາມນີ້ມີລາຍຊື່ທາງເລືອກບາງຢ່າງ, ແຕ່ສ່ວນຫຼາຍແມ່ນລາຍຊື່ວິທີທີ່ທ່ານສາມາດປ້ອງກັນຄວາມເປັນສ່ວນຕົວຂອງທ່ານໃນຂະນະທີ່ຍັງໃຊ້ YouTube ຢູ່, ເຖິງແມ່ນວ່າມັນຍັງເຮັດ ໜ້າ ທີ່ ຕຳ ໜິ ວິພາກວິຈານຢູ່.

[ການຄົ້ນຄ້ວາແບບ Virtual Machine ຂອງໂທລະສັບ Android] (https://github.com/seanpm2001/Degoogled_Android_Phone_VM_Research) - ເປັນສ່ວນ ໜຶ່ງ ທີ່ບໍ່ເປັນທາງການຂອງການໂຄສະນາ degoogleled ຂອງຂ້ອຍ, ນີ້ແມ່ນໂຄງການ ສຳ ລັບການຄົ້ນຄວ້າສ່ວນຕົວກ່ຽວກັບການສ້າງເຄື່ອງ virtual virtual ແບບເສີຍໆທີ່ມີປະໂຫຍດຢ່າງເຕັມສ່ວນ.

[ຢຸດ Googling - ເປັນຫຍັງທ່ານຄວນຢຸດໃຊ້ Google Search] (https://github.com/seanpm2001/Stop-Googling--Why-you-should-stop-using-Google-Search) - ບົດຂຽນຫຼັກອີກອັນ ໜຶ່ງ ໃນການໂຄສະນາ degoogle ຂອງຂ້ອຍ , ການ ກຳ ນົດເປົ້າ ໝາຍ ຂອງເຄື່ອງຈັກຊອກຫາຂອງ Google, ແລະສະແດງປະຫວັດຄວາມບົກຜ່ອງຂອງມັນ, ບວກກັບລາຍຊື່ທາງເລືອກຕ່າງໆ (ເຊັ່ນ DuckDuckGo, Ecosia, ແລະອື່ນໆ)

ເຖິງແມ່ນວ່າຫ້ອງຮຽນສ່ວນໃຫຍ່ແມ່ນໃຊ້ Scratch, ນີ້ແມ່ນບ່ອນທີ່ຄວາມສົນໃຈຂອງຂ້ອຍສູງ. ຂ້ອຍເປັນຜູ້ໃຊ້ Scratch ໃຫຍ່, ແລະໃນທີ່ສຸດກໍ່ໄດ້ເຂົ້າໄປໃນການຂຽນໂປແກຼມທີ່ແທ້ຈິງຫລັງຈາກໄດ້ປ່ຽນໄປ Python ໃນຊັ້ນຮຽນວິທະຍາສາດຄອມພິວເຕີ AP ທຳ ອິດຂອງຂ້ອຍ. ຂ້ອຍກໍ່ມັກ Python, ແລະຂ້ອຍມີຄວາມຮູ້ທີ່ສຸດໃນມັນອອກຈາກພາສາການຂຽນໂປແກຼມອື່ນ (ພາສາທີ່ຂຽນແລະເຄື່ອງ ໝາຍ (ເຊັ່ນ HTML ຫຼື Markdown) ບໍ່ໄດ້ລວມຢູ່ໃນບົດສະຫຼຸບນີ້)

### ທ່ານສາມາດຂຽນໂປແກຼມໃນ Java ໄດ້ດີເທົ່າໃດ

ຂ້າພະເຈົ້າໄດ້ຮຽນຊັ້ນ AP Java, ແຕ່ຂ້າພະເຈົ້າຍັງຂາດຂັ້ນຕອນຫລາຍໆຂັ້ນຕອນເພື່ອໃຫ້ໄດ້ເຕັມຮູບແບບ. ຂ້ອຍໄດ້ມີແນວຄວາມຄິດຫຼາຍຢ່າງກ່ຽວກັບມັນ, ແລະຫ້ອງຮຽນນີ້ໄດ້ຄຸ້ນເຄີຍຂ້ອຍກັບ Java ຂ້ອນຂ້າງຂ້ອນຂ້າງ, ພ້ອມກັບການໃຊ້ IDEs (Eclipse) ຂ້ອຍບໍ່ຄ່ອຍດີປານໃດກັບ Java, ແຕ່ຂ້ອຍສາມາດເຮັດບາງສິ່ງພື້ນຖານ, ບໍ່ແມ່ນພຽງແຕ່ໂປຣແກມ Hello World ເທົ່ານັ້ນ. ຂ້ອຍບໍ່ສາມາດຂຽນໂປຼແກຼມທັງ ໝົດ ໃນ Java ໄດ້, ແຕ່ຂ້ອຍຮູ້ສຶກຄືກັບຄວາມຮູ້ຂອງຂ້ອຍກ່ຽວກັບພື້ນຖານ Java ແລະລະດັບປານກາງແມ່ນ 2,8%

### ເປັນຫຍັງທ່ານໃຊ້ຫລາຍພາສາການຂຽນໂປແກຼມທີ່ແຕກຕ່າງກັນ

ຂ້ອຍຮູ້ຈັກສອງສາມຄົນ, ແຕ່ບໍ່ສາມາດເຮັດຫຍັງໄດ້ຫຼາຍກວ່າພື້ນຖານທີ່ສຸດ. ພາສາທີ່ຂ້ອຍເກັ່ງມີ Python, BASH, HTML5, ແລະ Java.

### ເປັນຫຍັງທ່ານມີໂຄງການຮູບພາບ? ມັນບໍ່ໄດ້ເອົາຊະນະຈຸດຂອງ GitHub

Wikipedia ແມ່ນ ໜຶ່ງ ໃນເວັບໄຊທ໌ທີ່ຂ້ອຍໃຊ້ຫຼາຍທີ່ສຸດ, GitHub ເປັນອັນດັບສອງ. ຂ້ອຍເຫັນວ່າ Wikipedia ເປັນຊັບພະຍາກອນທີ່ດີ, ແລະເປັນສິ່ງທີ່ດີສຳ ລັບການຄົ້ນຄວ້າຂຽນໂປແກຼມ, ເຄມີສາດ, ປະຫວັດສາດແລະອື່ນໆ. ຂ້ອຍຕັ້ງໃຈເລີ່ມບໍລິຈາກເງິນໃຫ້ Wikipedia ທຸກໆເດືອນ.

ຂ້ອຍມັກມີຄວາມສົມດຸນໃນຊີວິດຂອງຂ້ອຍ, ແລະມີເວລາພັກຜ່ອນຈາກການຂຽນໂປແກຼມ. ຂ້ອຍມີແນວຄວາມຄິດ ໃໝ່ໆ ໃນຊ່ວງເວລານີ້ເຊັ່ນກັນ, ແລະເຮັດວຽກອື່ນໆ, ເຊັ່ນວ່າວຽກເຮືອນຢູ່ອ້ອມເຮືອນ.

### ເປັນຫຍັງເຈົ້າຈຶ່ງຫຼີ້ນເກມເດັກນ້ອຍ

ຂ້ອຍຫຼິ້ນເກມທີ່ມີເປົ້າ ໝາຍ ທົ່ວໄປຂອງເດັກນ້ອຍ, ແຕ່ຍັງເປັນເກມທີ່ໃຊ້ເວລາໃນອະດີດ. ຕົວຢ່າງລວມມີຮ້ານອາຫານ / ເຂົ້າ ໜົມ ປັງ / ຮ້ານຂາຍສັດລ້ຽງ / ແຟຊັ່ນ / ເລື່ອງກະສິ ກຳ / ເມືອງ, ແລະຍົນ Merge. ເກມເຫຼົ່ານີ້ສາມາດໃຊ້ໄດ້ທັງເດັກນ້ອຍແລະຜູ້ໃຫຍ່. ຂ້ອຍໄດ້ຫຼີ້ນເກມເຫຼົ່ານີ້ໃນໄວເດັກຂອງຂ້ອຍ, ແລະມີຄວາມຜູກພັນກັບພວກເຂົາ. ບາງສ່ວນຂອງພວກເຂົາແມ່ນພຽງແຕ່ມ່ວນແທ້ໆໃນການເລີ່ມຕົ້ນ.

## ສະຖານະພາບຊອບແວ

ທຸກໆຜົນງານຂອງຂ້ອຍແມ່ນບໍ່ມີຂໍ້ ຈຳ ກັດບາງຢ່າງ. DRM (** D ** igital ** R ** estrictions ** M ** ມີສ່ວນພົວພັນ) ບໍ່ມີຢູ່ໃນວຽກງານໃດໆຂອງຂ້ອຍ.

! [DRM-free_label.en.svg] (DRM-free_label.en.svg)

ສະຕິກເກີນີ້ໄດ້ຮັບການສະ ໜັບ ສະ ໜູນ ຈາກມູນນິທິ Software Free. ຂ້ອຍບໍ່ເຄີຍຕັ້ງໃຈປະກອບ DRM ໃນວຽກຂອງຂ້ອຍ.

ຂ້ອຍ ກຳ ລັງໃຊ້ຕົວຫຍໍ້ວ່າ "ການຄຸ້ມຄອງຂໍ້ ຈຳ ກັດດິຈິຕອລ" ແທນທີ່ຈະຮູ້ຈັກ "ການຄຸ້ມຄອງສິດທິທາງດ້ານດິຈິຕອລ" ທີ່ເປັນທີ່ຮູ້ຈັກກັນທົ່ວໄປເນື່ອງຈາກວ່າວິທີການທົ່ວໄປຂອງການແກ້ໄຂມັນບໍ່ຖືກຕ້ອງ, ບໍ່ມີສິດກັບ DRM. ການສະກົດ ຄຳ ວ່າ "ການຄຸ້ມຄອງຂໍ້ ຈຳ ກັດດິຈິຕອລ" ແມ່ນຖືກຕ້ອງກວ່າເກົ່າ, ແລະໄດ້ຮັບການສະ ໜັບ ສະ ໜູນ ຈາກ [Richard M. Stallman (RMS)] (https://en.wikipedia.org/wiki/Richard_Stallman) ແລະ [Free Software Foundation (FSF)] ( https://en.wikipedia.org/wiki/Free_Software_Foundation)

ພາກນີ້ແມ່ນໃຊ້ເພື່ອປູກຈິດ ສຳ ນຶກ ສຳ ລັບບັນຫາຕ່າງໆກັບ DRM, ແລະເພື່ອປະທ້ວງມັນ ນຳ ອີກ. DRM ມີຂໍ້ບົກຜ່ອງໂດຍການອອກແບບແລະເປັນໄພຂົ່ມຂູ່ທີ່ ສຳ ຄັນຕໍ່ຜູ້ໃຊ້ຄອມພິວເຕີ້ແລະເສລີພາບໃນຊອບແວ.

ເຄຼດິດຮູບພາບ: [defectivebydesign.org/drm-free/... ຄົບ(https://www.defectivebydesign.org/drm-free/how-to-use-label)

> VideoLan / - \ - ຂ້ອຍຢາກທົດລອງແລະຂຽນເອກະສານ ສຳ ລັບ VideoLan, ຂ້ອຍເກັ່ງຫລາຍໃນການຂຽນເອກະສານ, ຍ້ອນວ່າຂ້ອຍມັກເຮັດມັນ, ແລະຂ້ອຍເຮັດມັນເກືອບທຸກໆມື້, ທຸກໆມື້. ປະຈຸບັນ, VideoLan ບໍ່ສາມາດມີນັກຂຽນເອກະສານເນື່ອງຈາກອາສາສະ ໝັກ ມີ ຈຳ ນວນ ໜ້ອຍ. ຖ້າໄດ້ຮັບອະນຸຍາດ, ຂ້ອຍຈະຂຽນເອກະສານ ສຳ ລັບທຸກໆ VLC ທີ່ບໍ່ມີມັນ (ເປັນພາສາອັງກິດເທົ່ານັ້ນ, ຂ້ອຍບໍ່ສາມາດເຮັດພາສາອື່ນໆໄດ້)

> Canonical (&) - ຂ້ອຍຢາກພະຍາຍາມແລະປະກອບສ່ວນໃຫ້ກັບ Linux ໃນທາງທີ່ຂ້ອຍສາມາດເຮັດໄດ້. ປະຈຸບັນເຮັດວຽກຢູ່ Canonical ແມ່ນທາງເລືອກ ໜຶ່ງ, ຂ້ອຍບໍ່ຮູ້ຢ່າງແນ່ນອນວ່າຂ້ອຍຈະເຮັດຫຍັງຢູ່ບ່ອນນີ້, ແຕ່ຂ້ອຍຫວັງວ່າຂ້ອຍຈະເຮັດຫຍັງໄດ້.

> ມູນນິທິ K (K) - KDE ຈະເປັນພື້ນຖານທີ່ດີໃນການເຮັດວຽກ. ຂ້ອຍຮັກໂຄງການເປີດໄຟເບົາທີ່ສຸດຂອງພວກເຂົາ, ແລະຂ້ອຍກໍ່ສາມາດຊ່ວຍຫຼຸດຜ່ອນອາການເຈັບຕາ. ຖ້າວຽກມີຢູ່ໃນພາສາທີ່ຂ້ອຍມີຄວາມຮູ້ຢ່າງ ໜ້ອຍ 40%, ຂ້ອຍສາມາດເຮັດວຽກໄດ້ທີ່ນີ້.

> ອົງກອນ GNOME (G) - ຖ້າເປັນໄປໄດ້, ຂ້ອຍສາມາດພະຍາຍາມແລະ ກຳ ນົດການປ່ຽນແປງບາງຢ່າງເພື່ອຫຼຸດຜ່ອນການ ນຳ ໃຊ້ຫນ່ວຍຄວາມ ຈຳ ທີ່ໃຫຍ່ຫຼວງຂອງ GNOME, ເຊິ່ງມັນເຄີຍຕິດຢູ່ກັບສິ່ງທີ່ ຈຳ ເປັນ ສຳ ລັບຂ້ອຍ. ມັນຍັງດີກວ່າການ ນຳ ໃຊ້ຫນ່ວຍຄວາມ ຈຳ ຂອງ Windows 10s, ການເຮັດວຽກທີ່ບໍ່ເຮັດວຽກເທິງ Windows 10 ຈະໃຊ້ RAM ຂອງຂ້ອຍເຄິ່ງ ໜຶ່ງ, ບໍ່ວ່າຂ້ອຍຈະມີ (4, 8, 16 GB) ໃນຄອມພິວເຕີ້ Linux ທຳ ອິດຂອງຂ້ອຍ, ມັນເຄີຍຢູ່ທີ່ຕ່ ຳ ກວ່າ 1.8 gigabytes ເມື່ອໃດ ບໍ່ເຮັດວຽກ (ຍັງນັບຕິດຕາມກວດກາລະບົບ)

### Linux

ຂ້ອຍເປັນແຟນພັດລົມຂອງ Windows up ຈົນເຖິງ Windows 10. ຂ້ອຍເລີ່ມພະຍາຍາມປ່ຽນໄປໃຊ້ Linux ຕັ້ງແຕ່ປີ 2018 ເຖິງປີ 2020. ໃນວັນທີ 9 ເດືອນກໍລະກົດປີ 2020, ສຸດທ້າຍຂ້ອຍໄດ້ປ່ຽນໄປໃຊ້ Linux, ໄດ້ຮັບຄອມພິວເຕີ Dell ລຸ້ນ ທຳ ອິດຂອງຂ້ອຍກັບ Linux ຕິດຕັ້ງໄວ້ກ່ອນ.

! [20200709_124359.jpg] (/ ສື່ / 20200709_124359.jpg)

ລະບົບປະຕິບັດການຂອງຂ້ອຍ ທຳ ອິດໃນລະບົບ Linux (ນອກ ເໜືອ ຈາກ Android 1.6) ແມ່ນ Ubuntu 20.04. ຂ້ອຍເປັນຜູ້ໃຊ້ Linux ທີ່ມີຄວາມພູມໃຈຫຼາຍ, ຍ້ອນວ່າຂ້ອຍບໍ່ໄດ້ ຈຳ ກັດການປັບປຸງການບັງຄັບໃຊ້ຂອງຂ້ອຍ, ຊອບແວທີ່ເປັນເຈົ້າຂອງແລະການອະນຸຍາດ, ການຂາດການຄວບຄຸມແລະການປັບແຕ່ງ, ຄວາມຢ້ານກົວຂອງຄວາມບໍ່ສະຖຽນລະພາບແລະບັນຫາອື່ນໆທີ່ຂ້ອຍມີກັບ Windows 10. ໃນວັນທີ 4 ເດືອນມີນາ 2021, ຂ້ອຍ ຂ້ອຍຍັງໃຊ້ Linux ຢູ່ໃນແຕ່ລະມື້, ແຕ່ຄວາມຢ້ານກົວຂອງ Blue Screens ຍັງບໍ່ ໝົດ, ບາງຄັ້ງກໍ່ຍັງຢ້ານວ່າຂ້ອຍຈະມີ ໜ້າ ຈໍສີຟ້າເມື່ອຂ້ອຍຮູ້ວ່າມັນບໍ່ເປັນໄປໄດ້ທາງດ້ານເຕັກນິກໃນ Ubuntu (ເວັ້ນເສຍແຕ່ວ່າເຈົ້າຈະມີໂປແກຼມທີ່ເຮັດໃຫ້ມັນຜິດ) ໃນເບື້ອງຕົ້ນແມ່ນການຕິດຕັ້ງ Fedora 32 ໃສ່ແລັບທັອບຂອງຂ້ອຍເມື່ອຂ້ອຍໄດ້ຮັບແຕ່ຂ້ອຍໄດ້ຕັດສິນໃຈໃຫ້ Ubuntu ກ່ອນ. ເຖິງຕອນນີ້, ຂ້ອຍມັກມັນ, ຍົກເວັ້ນ 2 ຢ່າງ: Snaps ເປັນເຈົ້າຂອງ, ແລະຄວາມລ້າໆຂອງ GNOME 3.x (ທຽບກັບລຸ້ນກ່ອນ ໜ້າ ນີ້ແລະສະພາບແວດລ້ອມ desktop ອື່ນໆ, ຄືກັບທີ່ຂ້ອຍຕ້ອງການແຕ່ບໍ່ໄດ້ຕິດຕັ້ງເທື່ອ; KDE)

## ການຍື່ນສະເຫນີ

### ການຖ່າຍຮູບ

ຂ້ອຍເປັນຊ່າງຖ່າຍຮູບລະດັບກາງ, ແລະສາມາດຖ່າຍຮູບທີ່ດີແທ້ໆໂດຍບໍ່ ຈຳ ເປັນຕ້ອງ ໝູນ ຮູບພາບເດີມເພື່ອເຮັດໃຫ້ມັນດີຂື້ນ (ນອກ ເໜືອ ຈາກເວລາທີ່ຂ້ອຍເອົານິ້ວໂປ້ໃສ່ ໜ້າ ຈໍໂດຍບັງເອີນ) ຂ້ອຍສາມາດຖ່າຍຮູບໃຫ້ເຈົ້າເມື່ອຕ້ອງການ. ກ້ອງຖ່າຍຮູບປະຈຸບັນຂອງຂ້ອຍແມ່ນ 4K (2160p, ຫຼື 2K) 60 ເຟຣມຕໍ່ວິນາທີ.

### ລອຍນໍ້າ

ຂ້ອຍມັກລອຍນໍ້າ. ເມື່ອຂ້ອຍມີສະລອຍນ້ ຳ ຫລືອ່າງຮ້ອນທີ່ມີໃຫ້ຂ້ອຍ, ຂ້ອຍມັກລອຍຢູ່ໃນນັ້ນ.

### ຫຼີ້ນເກມ

ຂ້ອຍເປັນຜູ້ຫຼິ້ນເກມວີດີໂອທີ່ມັກ, ມັກຫຼີ້ນເກມວີດີໂອຕ່າງໆເພື່ອຖ່າຍທອດເວລາ, ຜ່ອນຄາຍ, ມີຄວາມມ່ວນ, ແລະມີແນວຄິດ ໃໝ່ໆ. ຂ້ອຍຍັງເປັນຜູ້ຫຼີ້ນເກມບັດ, ແຕ່ຂ້ອຍຕ້ອງການຄວາມຊ່ວຍເຫລືອພິເສດໃນການຫຼີ້ນເກມບັດສ່ວນໃຫຍ່ຍ້ອນເສັ້ນໂຄ້ງການຮຽນຮູ້ແລະຄວາມຮູ້ທີ່ ຈຳ ກັດຕໍ່ເກມບັດ.

### ອອກແບບກາບຟິກ

ຂ້າພະເຈົ້າເປັນຜູ້ອອກແບບກຣາບຟິກລະດັບການເຂົ້າ - ອອກແບບໂດຍມີຄວາມມັກຫລາຍ ສຳ ລັບການອອກແບບຮູບຊົງ 3D ແລະຮູບຊົງ. ຂ້ອຍໄດ້ເອົາແວ່ນຕາອອກແບບກຣາຟິກ ຈຳ ນວນ ໜຶ່ງ ແລະສາມາດສ້າງຮູບພາບຄອມພິວເຕີໂດຍໃຊ້ GIMP, MS-PAINT, Blender 2.79, InkScape, Adobe PhotoShop CS6, 2015, CC 2017, Adobe Animate CS6, 2015, CC 2017, Adobe InDesign CC 2017, Adobe Illustrator CC 2017, BYOB 1.x, ແລະການຂູດ 2.

### ພາສາສາດ

ຂ້ອຍເປັນນັກພາສາສາດຕັ້ງແຕ່ໄວເດັກ. ຂ້ອຍຕະຫລົກປ່ຽນຊື່ກາງຂອງຂ້ອຍມາເປັນ `ພາສາ` ໃນໄວເດັກຂອງຂ້ອຍ, ແຕ່ຂ້ອຍຄິດວ່າມັນຈະ ເໝາະ ກັບຊື່ກາງອັນດັບ 2.

ຂ້ອຍມັກເບິ່ງແລະສຽງຂອງພາສາຕ່າງໆບໍ່ວ່າຈະເປັນ ຄຳ ເວົ້າຂອງມະນຸດຫລື ຄຳ ແນະ ນຳ ກ່ຽວກັບເຄື່ອງຈັກ. ແຕ່ໂຊກບໍ່ດີ, ຂ້ອຍບໍ່ສາມາດຂຽນພາສາອື່ນນອກ ເໜືອ ຈາກພາສາອັງກິດ, Python, HTML, ຫລືເຄື່ອງ ໝາຍ.

### ປະຫວັດຄວາມເປັນມາ

ຂ້ອຍແມ່ນຄວາຍປະຫວັດສາດ, ໂດຍມີຄວາມຮູ້ກ່ຽວກັບປະຫວັດສາດທີ່ມີປະຫວັດສາດມາຫຼາຍກວ່າ 13,8 ພັນລ້ານປີກ່ອນ (Big Bang) ຈົນເຖິງປະຈຸບັນ. ຂ້ອຍມັກຮຽນຮູ້ກ່ຽວກັບປະຫວັດສາດ, ແລະຂ້ອຍມີຄວາມຮູ້ກ່ຽວກັບການເລື່ອກສານແລະເຫດການໃຫຍ່ໆ, ລວມທັງຄວາມຮູ້ທີ່ກວ້າງຂວາງກ່ຽວກັບສິ່ງຕ່າງໆນອກ ເໜືອ ຈາກສົງຄາມໂລກຄັ້ງທີ 1 ຫຼືສົງຄາມໂລກຄັ້ງທີ 2 (ແຕ່ຂ້ອຍຍັງຮູ້ຫຼາຍກ່ຽວກັບສົງຄາມໂລກຄັ້ງທີ 2) ສຳ ລັບໂລກ ປະຫວັດສາດ, ຄວາມຮູ້ຂອງຂ້ອຍກັບຄືນສູ່ປີ 8000 BCE ໃນລາຍລະອຽດຢ່າງກວ້າງຂວາງ, ຫຼຸດລົງເຖິງ 27000 BCE ໃນລາຍລະອຽດ ໜ້ອຍ, ແລະສ່ວນຫຼາຍແມ່ນເຫດການດ້ານທໍລະນີສາດກ່ອນປີ 27000 BCE. ຂ້ອຍມັກຮຽນຮູ້ປະຫວັດສາດທຸກປະເພດ, ແລະບໍ່ພຽງແຕ່ປະຫວັດສາດຂອງສົງຄາມເທົ່ານັ້ນ.

ຂ້ອຍເປັນ ໜີ້ ຄວາມສົນໃຈໃນປະຫວັດສາດກັບຄູສອນປະຫວັດສາດຊັ້ນສູງຄົນ ທຳ ອິດຂອງຂ້ອຍ, ຜູ້ທີ່ມີຄວາມກະຕືລືລົ້ນແທ້ໆກ່ຽວກັບສິ່ງທີ່ລາວ ກຳ ລັງສອນ, ແລະເປັນພຽງຄູສອນປະຫວັດສາດທີ່ ໜ້າ ເກງຂາມແລະ ໜ້າ ຕາດີ. ແນ່ນອນວ່າຂ້ອຍຈະເຮັດໃຫ້ລາວເປັນຄູສອນປີ ໜຶ່ງ ສຳ ລັບປີຂອງໂຮງຮຽນມັດທະຍົມຕອນປາຍ. ຊັ້ນປະຫວັດສາດທີ່ຜ່ານມາບໍ່ເຄີຍເຮັດໃຫ້ຂ້ອຍເຂົ້າໄປໃນຫົວເລື່ອງຂອງປະຫວັດສາດ, ແຕ່ວ່າຫ້ອງຮຽນນີ້ກໍ່ໄດ້ເປີດໃຈຂອງຂ້ອຍສູ່ປະຫວັດສາດ.

### ຊີວະວິທະຍາທາງທະເລ

ຂ້າພະເຈົ້າບໍ່ດົນມານີ້ໄດ້ກັບຄືນສູ່ຊີວະສາດສັດທະເລໃນວັນທີ 23 ມີນາ 2021, ແລະໄດ້ຮຽນວິຊານີ້ບໍ່ເຕັມເວລາອີກເທື່ອ ໜຶ່ງ. ຂ້າພະເຈົ້າເຫັນວ່າມັນມີຄວາມສະຫງົບແລະ ໜ້າ ເສົ້າໃຈແທ້ໆ.

### ການລົງທືນ

ຂ້ອຍຮັກສາວາລະສານປະ ຈຳ ວັນ. ຂ້າພະເຈົ້າໄດ້ສະເຫວີຍສິ່ງນີ້ ສຳ ລັບສ່ວນໃຫຍ່ຂອງເດັກນ້ອຍຂອງຂ້ອຍ, ຈົນກ່ວາຕິດຢູ່ກັບມັນທຸກໆມື້ນັບຕັ້ງແຕ່ວັນທີ 26 ເດືອນກັນຍາປີ 2016.

## ທີມສີຟ້າ

ຂ້ອຍມີ 2 ທີມ ສຳ ລັບລະບົບອົງກອນຂອງຂ້ອຍ. ທີມສີຟ້າແມ່ນ ສຳ ລັບວຽກງານທາງວິຊາການ (ດີຈີຕອນ, ອະນາລັອກ, ຮາດແວ, ຊອບແວ) ແລະທີມສີຂຽວແມ່ນ ສຳ ລັບວຽກງານສິ່ງແວດລ້ອມ, ແລະຊີວະວິທະຍາ. ຂ້ອຍເປັນສ່ວນ ໜຶ່ງ ຂອງທີມສີຟ້າ, ແຕ່ຂ້ອຍຍັງເປັນສ່ວນ ໜຶ່ງ ຂອງທີມສີຂຽວ.
